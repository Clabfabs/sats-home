Frequently Asked Questions

1. What are the differences between the three ways to access SATS?

SATS Web Application (sats-web):
An easy-to-use web interface to generate auction instances (i.e., value files) directly in your browser, without having to install anything. The web interface allows for very light parametrization of the value models.
SATS Command Line Tool (sats-clt):
An executable jar, allowing the generation of auction instances (i.e., value files) in your command line. The available options allow for some parametrization of the generated value functions, yet not quite as much as sats-core.
SATS API (sats-core / sats-opt):
Java libraries to use in your own java code. Via the sats-core API, you gain access to the full feature set of SATS, including the ability to change all parameters of the value models, requesting values for the bundles on the fly, etc.
If you need to solve the winner determination problem for a generated auction instance, you can use sats-opt API, which extends the sats-core API.

2. For which value models does SATS already contain the MIP formulation to solve the winner determination problem?

Currently, SATS includes a MIP for the following value models:

  • Multi-Region Value Model (MRVM)
  • Single-Region Value Model (SRVM)
  • Local Synergy Value Model (LSVM)
  • Global Synergy Value Model (GSVM)
