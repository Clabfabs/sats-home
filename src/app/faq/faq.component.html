What are the differences between the three ways to access SATS?

SATS Core API (sats-core):
A java library to use in you own java code. It includes a wide variety of spectrum value models and allows to configure and to generate new test problem instances from these value models. This includes generating value functions in different bidding languages. Should you require to solve the winner determination problems of the generated problem instances, we recommend you to use sats-opt, which extends sats-core.
SATS Web Application (sats-web):
A very easy to use web interface to generate test problem instances (i.e., value files) directly in your browser, without having to install anything. There are some parameters to parameterize the generated value function, yet not quite as many as sats-core.
SATS Command Line Tool (sats-clt):
An executable jar, allowing to generate test problem instances (i.e., value files) in your command line. The available options to configure the generated problems are limited. There are some parameters to parameterize the generated value function, yet not quite as many as sats-core.

For which value models does SATS already contain the MIP formulation?

Currently, SATS includes a MIP for the following value models:

  • Multi-Region Value Model (MRVM)
  • Single-Region Value Model (SRVM)
  • Global Synergy Value Model (GSVM)
